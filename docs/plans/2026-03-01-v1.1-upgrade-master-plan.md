# LinkedIn Post Generator v1.1 - Master Upgrade Plan

## 1. Context & Strategic Vision
**Why are we doing this?**
The current v1.0 system is likely a single-user tool dependent on manual configuration and local files. The v1.1 upgrade transforms this into a **Multi-Tenant SaaS Platform**. 

**Key Drivers for Upgrade:**
1.  **Scalability (Multi-Tenancy):** Moving from single-user scripts to a database-backed architecture (`user_id` isolation) allows the system to serve multiple users securely.
2.  **Automation (Firecrawl & Ingestion):** Eliminating manual copy-pasting of brand assets and manual parsing of CSVs. The system will auto-extract brand identity and ingest LinkedIn data dumps.
3.  **Intelligence (RAG & CRM):** Shifting from generic templates to hyper-personalized content. The "Voice Engine" uses Vector RAG to "speak" like the user, and the CRM logic identifies *who* to engage with based on behavioral analysis.
4.  ** robustness:** Replacing fragile ad-hoc scripts with a structured 3-layer architecture (Directive -> Orchestration -> Execution).

## 2. System Architecture
- **Backend:** Python (FastAPI/Scripts)
- **Database:** 
  - **Relational:** SQL (SQLite for dev, PostgreSQL/Supabase for prod) for User Profiles, Contacts, Posts.
  - **Vector:** pgvector (or local FAISS/Chroma for dev) for RAG Voice Engine.
- **Frontend:** Vanilla JS + HTML/CSS (Lightweight, fast).
- **External APIs:** Firecrawl (Scraping), Gemini (AI/LLM).

## Phase 3: Network Intelligence - Data Ingestion
**Goal:** Ingest raw LinkedIn data dumps (ZIP exports) into the structured database.

### Step 3.1: ZIP & CSV Parsing Logic
- **Action:** Create `execution/linkedin_parser.py`.
- **Logic:** 
    - Accept a ZIP file path.
    - Extract to `.tmp/`.
    - Locate `Connections.csv`, `messages.csv`, `Profile.csv` (handling nested `raw/` folders).
    - Parse CSVs using Pandas.

### Step 3.2: Data Normalization & Upsert
- **Action:** Implement "Upsert" logic. If a contact exists (by LinkedIn URL/URN), update them; otherwise insert.
- **Performance:** Use bulk inserts for thousands of connections to avoid slow row-by-row processing.

---

## Phase 4: Network Intelligence - Behavioral CRM
**Goal:** Turn raw contacts into actionable segments (Leads, Friends, Ghosts).

### Step 4.1: Message History Analyzer
- **Action:** Create `execution/message_analyzer.py`.
- **Logic:** Group messages by `thread_id` or `participant`.
- **Rules:** 
    - 0 replies = "Cold"
    - <3 replies = "Superficial"
    - >3 replies + recent = "Warm"
    - Last msg from User > 14 days ago = "Ghosted"

### Step 4.2: AI Intent Classification (Gemini)
- **Action:** Create `execution/intent_classifier.py`.
- **Logic:** Send the last 5 messages of "Warm" threads to Gemini.
- **Prompt:** "Analyze this conversation. Is this a potential lead, a partner, or a recruiter? Summarize intent in 5 words."
- **Output:** Update `Contact` table with `ai_intent` and `ai_summary`.

---

## Phase 5: Voice Engine (RAG Setup)
**Goal:** Enable the AI to write *as* the user by referencing their past content.

### Step 5.1: Content Chunking
- **Action:** Create `execution/rag_manager.py`.
- **Logic:** Read `Profile.csv` (About section, Experience) and scraped website text. Split into chunks (e.g., 500 characters).

### Step 5.2: Vector Embedding & Storage
- **Action:** Integrate an embedding model (e.g., `text-embedding-3-small` or local).
- **Action:** Store embeddings in the Vector DB linked to `user_id`.

### Step 5.3: Retrieval Logic
- **Action:** Implement `query_user_voice(user_id, query)` function.
- **Logic:** Input "AI trends" -> Retrieve user's past posts/resume bullets about AI -> Return as context.

---

## Phase 6: Dynamic Content Generation
**Goal:** The final output - generating posts that combine Brand Voice + Network Intelligence.

### Step 6.1: Master Prompt Construction
- **Action:** Create `execution/prompt_engine.py`.
- **Logic:** 
    - Load Static Directive (e.g., `educational_article.md`).
    - Inject Dynamic Context: `{{BRAND_VOICE}}` (from DB), `{{RAG_CONTEXT}}` (from Phase 5), `{{TARGET_AUDIENCE}}` (from CRM).

### Step 6.2: Generation Execution
- **Action:** Call Gemini API with the Master Prompt.
- **Output:** Return Markdown content to the Frontend.
