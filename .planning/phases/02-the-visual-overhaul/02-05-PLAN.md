# Phase 2 Gap Closure: Studio State and Streaming Integration

## Objective
Address verification gaps in the Studio tab by implementing responsive mobile switching, wiring the generating state to global Zustand store, and implementing the Server-Sent Events (SSE) stream for real-time text generation in the Canvas component.

## Tasks

<tasks>
  <task id="studio_mobile_view" title="Implement mobile state toggle in Studio page">
    Update `frontend-next/app/studio/page.tsx` to conditionally show either the form or the canvas on mobile screens, instead of permanently hiding the canvas. Ensure there is a UI affordance to toggle back to the form after generation.
  </task>

  <task id="canvas_state_wire" title="Wire Canvas to global state" depends_on="studio_mobile_view">
    Update `frontend-next/components/studio/Canvas.tsx` to read `isGenerating` and content from the Zustand `useAppStore` instead of using hardcoded variables. Ensure skeletons render correctly when `isGenerating` is true.
  </task>

  <task id="canvas_sse_stream" title="Implement SSE streaming in Canvas" depends_on="canvas_state_wire">
    Implement a hook or logic in `frontend-next/components/studio/Canvas.tsx` to process Server-Sent Events (SSE) from the backend `/api/generate` endpoint, appending text chunks to the global state as they stream in. Ensure `isGenerating` toggles properly on completion.
  </task>
</tasks>

## Verification

### Must Haves
- `frontend-next/app/studio/page.tsx` must implement a toggle mechanism for mobile devices to switch between form and canvas.
- `frontend-next/components/studio/Canvas.tsx` must consume `isGenerating` from `useAppStore` to conditionally display the shimmering skeleton.
- `frontend-next/components/studio/Canvas.tsx` must contain logic to connect to an SSE stream and update the post content state.

### Testing Steps
1. Simulate mobile view. Submitting the form should transition the view to the Canvas.
2. In desktop view, verify the form is on the left and canvas on the right.
3. Submit the form; verify `isGenerating` becomes true and the skeleton appears immediately.
4. Verify the frontend attempts an SSE connection to stream text, appending incoming chunks.
